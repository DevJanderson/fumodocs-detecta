## 🎯 Como Contribuir

### 1. Documentação de Módulos
Para documentar um novo módulo:

1. Crie um arquivo `nome-do-modulo.md` em `docs/modules/`
2. Use o template base da documentação
3. Inclua todas as seções relevantes:
   - Visão Geral
   - Arquitetura
   - Funcionalidades
   - Implementação
   - Como Usar
   - Troubleshooting

### 2. Estrutura Padrão
```markdown
# 📰 Nome do Módulo - DETECTA

## 📋 Visão Geral
[Descrição do módulo]

## 🏗️ Arquitetura
[Estrutura de arquivos e tecnologias]

## 🎯 Funcionalidades
[Lista de funcionalidades principais]

## 🔧 Implementação
[Código e exemplos]

## 🛠️ Como Usar
[Instruções de uso]

## 🔍 Troubleshooting
[Problemas comuns e soluções]
```

### 3. Atualização do Índice
Após criar a documentação de um módulo, atualize este README.md marcando como ✅.

## 🎨 Padrões de Design

### Responsividade
- **Mobile First**: Abordagem mobile-first para todos os componentes
- **Breakpoints**: 640px (tablet), 1024px (desktop)
- **Grid Adaptativo**: 1 coluna (mobile) → 2 colunas (tablet) → 3 colunas (desktop)

### Cores ITpS
- **🔴 ITpS Red**: `#F5685D` - Covid, alertas críticos
- **🟢 ITpS Green**: `#59B852` - Sucesso, status positivo
- **🟠 ITpS Orange**: `#F5C644` - Dengue, avisos
- **🔵 ITpS Blue**: `#1069af` - Gripe, informações
